# Nick OPC 项目架构

## 一、整体架构

```
                                    Nick OPC 辅助管家
                                  一人公司专属 AI 助手
                                         │
      ┌────────────────────────────────────┼────────────────────────────────────┐
      │                                    │                                    │
      ▼                                    ▼                                    ▼
 飞书入口 (对话式)              Web Dashboard (可视化)              API 接口 (第三方)
      │                                    │                                    │
      └────────────────────────────────────┼────────────────────────────────────┘
                                         │
                                         ▼
                           LLM Gateway (大脑)
                      自然语言理解 → 任务解析
                                         │
         ┌─────────────────────────┼─────────────────────────┐
         │                         │                         │
         ▼                         ▼                         ▼
    任务路由 (层级调度)      知识库 (RAG)              数据存储 (企业数据)
```

## 三、核心模块

```
┌─────────────────────────────────────┐
│         核心脚本模块               │
├─────────────────────────────────────┤
│                                     │
│  opc_parser.py       (任务解析器)  │
│  opc_contract_detector.py (合同检测)│

```

## 四、数据流

```
用户输入 → 飞书/Web → LLM理解 → 任务解析 → Agent调度 → 执行 → 反馈 → 记录
```

## 五、技术栈

| 层级 | 技术 |
|------|------|
| 对话入口 | 飞书机器人 / WebSocket |
| AI 能力 | LLM Gateway / MiniMax / OpenRouter |
| 存储 | JSON文件 / 向量数据库 (待接入) |
| 部署 | Docker / 一键脚本 |

## 六、OPC 专属能力

| Agent | 能力 |
|-------|------|
| 财政部 | 税务计算、申报辅助 |
| 宗教裁判所 | 合同风险检测、合规检查 |
| 占星术士 | 政策监控、风险预警 |
| 工匠行时 | 政务平台对接 |
