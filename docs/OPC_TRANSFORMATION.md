# Nick 项目改造方案：OPC（一人公司）专属辅助管家

**版本**: 1.0  
**日期**: 2026-03-22

---

## 一、核心定位重构

### 从「通用企业级 Agent」→「OPC 专属辅助管家」

以 OPC 经营者为唯一核心，围绕"合规+效率+降成本"，提供"决策建议+流程简化+落地执行"的全链路辅助。

### 角色映射

| 原有 Agent | 改造后 OPC 专属职能 | 核心场景 |
|------------|---------------------|----------|
| 教皇（Pope）| OPC 经营者（用户）| 最终决策人 |
| 红衣主教团（Cardinal）| 事务规划师 | 拆解核心任务 |
| 主教团（Bishop）| 事务调度器 | 协调各专项 Agent |
| 宗教裁判所（Inquisition）| 合规风控 Agent | 税务风险、人格否认风险 |
| 财政部（Treasury）| 财税辅助 Agent | 记账、报税、优惠匹配 |
| 工匠行会（Guild）| 技术对接 Agent | 对接电子税务局/代账平台 |
| 占星术士（Astrologer）| 经营监控 Agent | 营收分析、政策提醒 |
| 枢机处（Cardinal Office）| 审核兜底 Agent | 关键操作二次核查 |

## 二、分阶段改造方案

### 阶段 1：MVP 落地（第 1 个月）

**目标**：解决 OPC 最痛的"税务申报+合规风险预警"

**核心功能**：
- 自然语言交互：用户输入"申报本月增值税" → 自动采集数据 → 生成报表 → 合规核查 → 申报
- 风险预警：税务申报截止日预警、优惠政策匹配、财产混同风险提醒

**停用 Agent**：
- 典礼部、人事部、骑士团（仅保留基础功能）

**保留 Agent**：
- 财政部（财税）
- 宗教裁判所（合规）
- 工匠行会（接口）
- 占星术士（监控）

**技术补充**：
- 自然语言→结构化任务转换模块
- 政务接口适配层（对接第三方代账平台）
- 政策知识库（实时同步税务局政策）

**合规兜底**：
- 明确免责声明
- 对接有资质第三方（不触碰红线）
- 审计日志留存

---

### 阶段 2：进阶优化（第 2-3 个月）

**目标**：覆盖 OPC 80% 高频事务

**新增功能**：
- 工商 Agent：注册核名、营业执照、变更登记、年报申报
- 合同 Agent：常用合同模板 + 智能风险核查
- 社保 Agent：增员/减员、缴费核算、申报缴纳

**技术优化**：
- OPC 统一数据中台（数据互通）
- 人机协同兜底（人工复核通道）
- 可视化 Dashboard

**风险兜底**：
- 引入服务保障险
- 占星术士每日政策爬取更新

---

### 阶段 3：成熟闭环（第 3-6 个月）

**目标**：商业化 + 全链路自动化

**商业变现**：
- 基础版（免费）：税务预警、简单报表
- 进阶版（包月）：全流程报税、合同审核
- 增值服务：线下代账/律所对接、融资对接

**技术深化**：
- 自我学习（基于用户反馈迭代）
- 多端适配（小程序/APP）
- 生态对接（银行流水、电子签章）

## 三、关键问题解决

| 核心问题 | 解决策略 |
|----------|----------|
| 场景与 Agent 能力不闭环 | 先聚焦税务/合规刚需场景，主教团统一调度 |
| 合规/法律风险 | 对接有资质第三方 + 免责声明 + 服务保险 |
| 多 Agent 协作效率低 | 复用消息队列/事件总线，搭建数据中台 |
| 用户体验差 | 自然语言交互 + 可视化 Dashboard |
| 商业闭环难 | 免费引流 + 进阶版收费 + 增值服务 |

## 五、注意事项

1. **不越权**：始终以用户为最终决策人，Agent 仅提供建议
2. **轻量迭代**：MVP 只解决税务+合规，验证后扩展
3. **政策敏感**：关注一人公司核心红线（财产独立核算、年度审计）
4. **成本控制**：复用现有模块，降低研发成本
