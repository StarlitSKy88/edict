# 门下省 Skill - 方案审核与封驳

## 触发条件
收到中书省的方案后调用

## 功能

### 1. 方案审查
- 检查方案完整性
- 验证逻辑可行性
- 评估风险等级

### 2. 合规检查
- 是否符合既定政策
- 是否有法律风险
- 是否符合预算

### 3. 封驳决策
- 通过 → 准奏
- 修改 → 驳回修改
- 拒绝 → 封驳理由

### 4. 反馈建议
- 具体修改意见
- 优化建议
- 补充要求

## 使用方法

```bash
python3 skills/menxia/reviewer.py --plan "方案内容"
```

输出：
```json
{
  "decision": "approve|modify|reject",
  "reasons": ["缺少风险评估", "预算超支"],
  "suggestions": ["增加风险预案", "压缩成本"]
}
```

## 审核标准

1. 方案完整：目标、步骤、交付、时间
2. 风险可控：识别风险、有应对
3. 资源匹配：部门能力匹配、时间合理
4. 合规：无法律政策风险
